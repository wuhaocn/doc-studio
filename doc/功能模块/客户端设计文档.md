# Memora 客户端设计文档

## 1. 项目概述

### 1.1 项目名称
Memora 客户端 - 在线文档管理系统桌面客户端

### 1.2 项目简介
Memora 客户端是 Memora 在线文档管理系统的桌面应用，基于 Tauri + Rust 技术栈，提供本地文件管理、文档管理和云端同步功能。客户端与 Java 后端服务无缝集成，支持离线工作和多端数据同步。

**核心功能**:
- **本地文件管理**: 浏览、编辑、组织本地文件和文件夹
- **文档管理**: 创建、编辑、存储本地文档
- **云端同步**: 与 Memora Java 后端服务同步数据
- **离线工作**: 支持离线状态下的文档编辑和管理
- **本地搜索**: 快速搜索本地文件和文档
- **文件预览**: 支持多种文件格式的预览

### 1.3 项目目标
- 提供原生桌面应用体验
- 实现本地文件和文档的高效管理
- 支持与云端系统的无缝同步
- 提供离线工作能力
- 优化性能和用户体验
- 支持多平台（Windows、macOS、Linux）

### 1.4 系统架构
```
┌─────────────────┐
│   客户端应用     │
│  memora-client  │
│  (Tauri+Rust)   │
└────────┬────────┘
         │
         └───► Memora 后端服务
                │
                │ (Java + Spring Boot)
                ▼
           数据库存储
```

## 2. 技术栈

### 2.1 核心技术栈
- **框架**: Tauri 1.5+（基于 Rust 和 Web 技术）
- **客户端后端**: Rust 1.70+（核心业务逻辑）
- **前端**: React 18 + TypeScript
- **服务端**: Java 21+（Spring Boot）
- **UI组件库**: Arco Design
- **文档编辑器**: Tiptap
- **HTTP客户端**: reqwest（Rust）/ Axios（前端）
- **构建工具**: Cargo（Rust）/ Vite（前端）/ Gradle（服务端）
- **代码规范**: ESLint + Prettier

## 4. 核心功能设计

### 4.1 本地文件管理

#### 4.1.1 文件浏览
- **功能**: 浏览本地文件和文件夹
- **实现**: 使用 Rust 原生文件系统 API 实现
- **UI设计**: 文件浏览器界面，支持树形结构和列表视图

#### 4.1.2 文件操作
- **功能**: 复制、移动、重命名、删除文件和文件夹
- **实现**: 使用 Rust 原生文件系统 API 实现
- **UI设计**: 右键菜单和工具栏操作

#### 4.1.3 本地搜索
- **功能**: 快速搜索本地文件和文档
- **实现**: 基于文件系统索引的搜索
- **UI设计**: 搜索栏和搜索结果列表

### 4.2 文档管理

#### 4.2.1 文档编辑
- **功能**: 创建和编辑文档，支持富文本编辑
- **实现**: 使用 Tiptap 富文本编辑器，支持多种编辑功能
- **UI设计**: 类似在线文档编辑器的界面，顶部工具栏，中间编辑区域

#### 4.2.2 文档版本管理
- **功能**: 查看和管理文档历史版本
- **实现**: 本地版本存储和云端版本同步
- **UI设计**: 版本历史侧边栏，显示版本列表和创建时间

#### 4.2.3 文档排序
- **功能**: 调整文档的显示顺序
- **实现**: 本地存储文档排序信息
- **UI设计**: 拖拽排序界面，支持上下移动文档

### 4.3 云端同步

#### 4.3.1 同步设置
- **功能**: 配置同步选项和频率
- **实现**: 通过同步服务配置
- **UI设计**: 同步设置界面，支持自动和手动同步

#### 4.3.2 同步状态
- **功能**: 显示同步状态和进度
- **实现**: 通过同步服务监控与 Java 后端服务的同步状态
- **UI设计**: 同步状态指示器和进度条

#### 4.3.3 冲突处理
- **功能**: 处理同步冲突
- **实现**: 冲突检测和解决机制
- **UI设计**: 冲突解决对话框

### 4.4 离线工作

#### 4.4.1 离线模式
- **功能**: 在离线状态下继续工作
- **实现**: 本地存储和缓存机制
- **UI设计**: 离线状态指示器

#### 4.4.2 离线操作
- **功能**: 记录离线状态下的操作
- **实现**: 操作日志和本地存储
- **UI设计**: 离线操作提示

### 4.5 文件预览

#### 4.5.1 预览功能
- **功能**: 预览多种文件格式
- **实现**: 基于文件类型的预览组件
- **UI设计**: 预览模态框，支持缩放和导航

#### 4.5.2 预览支持
- **功能**: 支持多种文件格式的预览
- **实现**: 集成不同的预览引擎
- **UI设计**: 文件预览界面

### 4.6 知识库管理

#### 4.6.1 知识库创建和编辑
- **功能**: 创建和编辑知识库
- **实现**: 本地存储和云端同步
- **UI设计**: 知识库表单，包含名称和描述字段

#### 4.6.2 知识库同步
- **功能**: 同步知识库到云端
- **实现**: 通过同步服务实现
- **UI设计**: 同步状态指示

### 4.7 资源管理

#### 4.7.1 资源上传和管理
- **功能**: 上传和管理资源文件
- **实现**: 本地存储和云端同步
- **UI设计**: 资源管理界面，支持拖放上传

#### 4.7.2 资源搜索
- **功能**: 搜索资源文件
- **实现**: 本地搜索和云端搜索
- **UI设计**: 资源搜索表单，显示搜索结果列表

## 10. 技术风险与应对

### 10.1 性能瓶颈
- **问题**: 大文件上传和处理可能导致性能问题
- **解决**: 实现文件分块上传，优化处理算法

### 10.2 兼容性问题
- **问题**: 不同浏览器的兼容性差异
- **解决**: 使用浏览器兼容性库，进行多浏览器测试

### 10.3 依赖管理
- **问题**: 依赖库版本更新可能导致兼容性问题
- **解决**: 锁定依赖版本，定期更新和测试
