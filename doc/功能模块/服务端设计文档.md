# Memora 服务端设计文档

## 1. 项目概述

### 1.1 项目名称
Memora 服务端 - 在线文档管理系统后端

### 1.2 项目简介
Memora 服务端是 Memora 在线文档管理系统的核心后端服务，提供文档存储、用户管理、权限控制、同步服务等核心功能。服务端与客户端和Web端无缝集成，支持多端数据同步和协作编辑。集成了香港大学HKUDS实验室开源的DeepTutor AI学习助手功能和OpenClaw技能化架构。

**核心功能**:
- **用户管理**: 注册、登录、权限控制
- **文档管理**: 创建、编辑、存储、版本控制
- **同步服务**: 与客户端和Web端的数据同步
- **协作功能**: 多用户实时协作编辑
- **存储管理**: 文件存储、元数据管理
- **搜索服务**: 文档内容搜索
- **AI服务**: DeepTutor AI学习助手功能
- **技能系统**: 基于OpenClaw的技能化架构
- **技能执行环境**: 隔离的容器环境，用于安全执行技能
- **技能市场**: 提供技能的分享、发现和安装功能
- **OpenClaw技能系统**: 技能管理、执行、市场和执行环境管理
- **DeepTutor AI学习助手**: 智能问答、可视化、出题等功能

### 1.3 项目目标
- 提供稳定、高效的后端服务
- 支持大规模文档存储和管理
- 实现多端数据同步
- 提供安全的用户认证和权限控制
- 支持实时协作编辑
- 优化系统性能和可靠性
- 集成OpenClaw技能化架构，支持本地执行和主动代理
- 集成DeepTutor AI学习助手功能

## 2. 技术栈

### 2.1 核心技术栈
- **语言**: Java 21+
- **Web框架**: Spring Boot 3.3.5+
- **数据库**: MySQL
- **缓存**: Redis
- **认证**: JWT
- **构建工具**: Gradle
- **消息队列**: RabbitMQ / RocketMQ（用于异步消息处理）
- **文件存储**: MinIO / OSS（用于文件存储）
- **实时通信**: WebSocket（用于协作功能）
- **容器技术**: Docker（用于技能执行环境隔离）
- **AI模块**: 
  - DeepTutor（香港大学HKUDS实验室开源AI学习助手）
  - OpenClaw（技能化架构框架）

### 2.2 核心依赖
- **spring-boot-starter-web**: Web框架
- **mybatis-plus-boot-starter**: ORM
- **spring-boot-starter-security**: 安全框架
- **jjwt**: JWT认证
- **redis-spring-boot-starter**: 缓存
- **spring-boot-starter-jdbc**: JDBC支持
- **spring-boot-starter-websocket**: WebSocket支持
- **rabbitmq-spring-boot-starter**: 消息队列
- **spring-boot-starter-data-redis**: Redis支持
- **swagger-ui**: API文档
- **knife4j**: API文档增强

## 4. 核心功能设计

#### 4.1.1 文档创建
- **功能**: 创建新文档
- **实现**: 文档元数据存储，内容存储
- **API**: `POST /api/documents`

#### 4.1.2 文档编辑
- **功能**: 编辑文档内容
- **实现**: 内容更新，版本控制
- **API**: `PUT /api/documents/{id}`

#### 4.1.3 文档版本管理
- **功能**: 管理文档历史版本
- **实现**: 版本快照，差异存储
- **API**: `GET /api/documents/{id}/versions`

#### 4.1.4 文档排序
- **功能**: 调整文档顺序
- **实现**: 更新文档排序字段
- **API**: `PUT /api/documents/sort`

#### 4.2.1 知识库创建

#### 4.2.2 知识库编辑
- **功能**: 编辑知识库信息
- **实现**: 更新知识库元数据
- **API**: `PUT /api/knowledge-bases/{id}`

#### 4.2.3 知识库统计
- **功能**: 获取知识库统计信息
- **实现**: 统计知识库中的文档和资源数量
- **API**: `GET /api/knowledge-bases/{id}/stats`

- **功能**: 编辑资源信息
- **实现**: 更新资源元数据

#### 4.3.3 资源搜索

## 7. 同步机制设计

### 7.1 同步流程
1. **客户端请求**: 客户端发送同步请求，包含上次同步时间和版本
2. **差异检测**: 服务端检测自上次同步以来的变化
3. **数据传输**: 服务端发送变化的数据给客户端
4. **客户端应用**: 客户端应用变化到本地
5. **确认同步**: 客户端确认同步完成

### 7.2 冲突处理
1. **冲突检测**: 检测到同一文档的并发修改
2. **冲突解决**: 根据策略解决冲突
   - **最后修改优先**: 使用最后修改的版本
   - **手动解决**: 提示用户选择
   - **合并**: 尝试自动合并更改
3. **冲突记录**: 记录冲突解决历史

### 7.3 离线同步
1. **离线操作**: 客户端记录离线状态下的操作
2. **操作日志**: 保存操作序列
3. **连接恢复**: 检测网络连接恢复
4. **批量同步**: 批量上传离线操作
5. **冲突处理**: 处理可能的冲突
